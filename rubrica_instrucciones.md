Examen Práctico 2
Objetivo:
El presente componente práctico tiene como finalidad integrar y aplicar los conocimientos adquiridos durante el curso, a través del desarrollo de una solución funcional que incluya el diseño, implementación e interacción con una base de datos.

Instrucciones:

Este trabajo parte del primer entregable del componente práctico y debe desarrollarse en tres fases:

Fases del Trabajo
Diseño Final del Modelo de Base de Datos
Afinar el modelo relacional inicial con base en la retroalimentación recibida.
Justificar mejoras estructurales (normalización, relaciones, restricciones).
Entregar el diagrama/modelo relacional final.
Implementación del Modelo en un Motor de Base de Datos
Implementar la base de datos en SQL Server u otro gestor aprobado.
Crear las estructuras necesarias: tablas, relaciones, claves primarias y foráneas, y procedimientos de estandarización para el modelo.
Cargar datos de prueba representativos del modelo de negocio.
Desarrollo de Interfaz Gráfica de Gestión
Desarrollar una aplicación con interfaz gráfica (desktop o web) que permita:
Registrar, consultar, actualizar y eliminar información (CRUD).
Interactuar directamente con la base de datos implementada.
Presentar al menos 4 consultas complejas que respondan a requerimientos del modelo de negocio.
La aplicación puede desarrollarse en cualquier lenguaje o entorno que el estudiante maneje (ej. Python + Tkinter / Java + Swing / .NET / HTML+PHP / otros).
Recomendaciones:
Asegúrese de documentar claramente su propuesta técnica y justificar las decisiones de diseño.
Pruebe su sistema antes de entregarlo: funcionalidad, validaciones, integridad de datos.
La interfaz debe ser clara, intuitiva y reflejar adecuadamente el modelo de datos.
Evite plantillas genéricas no relacionadas con el modelo trabajado.
Trabaje de forma autónoma y ética. El plagio o la copia entre grupos implicará calificación cero.
Rúbrica de evaluación (15 puntos):
Criterio de Evaluación	Puntaje Máximo
1. Modelo relacional final ajustado y justificado	3 puntos
2. Implementación correcta en el gestor de base de datos (estructuras, relaciones, datos de prueba)	4 puntos
3. Funcionamiento de la aplicación: operaciones CRUD sobre la base de datos	3 puntos
4. Consultas relevantes y funcionales (mínimo dos, con valor agregado)	2 puntos
5. Calidad técnica y usabilidad de la interfaz gráfica	2 puntos
6. Presentación clara del entregable, documentación mínima y cumplimiento de instrucciones	1 punto
